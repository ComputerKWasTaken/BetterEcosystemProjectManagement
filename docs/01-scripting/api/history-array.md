# History Array

> The history array contains the sequence of past actions in the adventure, providing scripts access to the narrative record.

## Overview

The `history` array stores all actions that have occurred in the adventure. Each element represents a single action—either player input or AI response. Scripts can read history to understand context, detect patterns, or analyze the narrative.

## Structure

```
history = [
  { text: string, type: string, rawText: string },
  { text: string, type: string, rawText: string },
  ...
]
```

## Action Object Properties

### text

**Type**: string

**Description**: The text content of the action

**Notes**: This is the primary text content to use

### type

**Type**: string

**Description**: The type of action

**Possible Values**:
| Type | Description |
|------|-------------|
| `'start'` | The first action of an adventure (from scenario prompt) |
| `'continue'` | An action generated by the AI |
| `'do'` | A Do action submitted by a player |
| `'say'` | A Say action submitted by a player |
| `'story'` | A Story action submitted by a player |
| `'see'` | A See action (image generation) |
| `'repeat'` | A retry/regeneration |
| `'unknown'` | Type could not be determined |

### rawText

**Type**: string

**Description**: Deprecated. Same as `text`. Included for backward compatibility.

## Array Order

Actions are stored chronologically:
- `history[0]` is the oldest action (typically 'start')
- `history[history.length - 1]` is the most recent action

## Accessing Recent History

### Most Recent Action
```
const lastAction = history[history.length - 1];
```

### Previous Action
```
const previousAction = history[history.length - 2];
```

### Last N Actions
```
const recentActions = history.slice(-5);  // Last 5 actions
```

## History During Each Hook

### onInput
- History contains all actions up to (but NOT including) the current player input
- The action being processed is in `text`, not yet in history

### onModelContext
- Same as onInput—current action not yet in history

### onOutput
- History still does not include the current AI response
- The AI output being processed is in `text`
- After onOutput completes, the response is added to history

## Common Patterns

### Check If First Turn
```
if (history.length === 0) {
  // Very first action, history is empty
}
```

### Find Last Player Action
```
const lastPlayerAction = [...history]
  .reverse()
  .find(a => ['do', 'say', 'story'].includes(a.type));
```

### Find Last AI Response
```
const lastAIResponse = [...history]
  .reverse()
  .find(a => a.type === 'continue');
```

### Search History for Pattern
```
const mentionedDragon = history.some(a => 
  a.text.toLowerCase().includes('dragon')
);
```

### Get Recent Text Combined
```
const recentText = history
  .slice(-5)
  .map(a => a.text)
  .join('\n');
```

## Read-Only Behavior

The history array is provided for reading. While JavaScript allows array modification, changes to history may not persist or may cause unexpected behavior. Use `state` for persistent data storage.

## Relationship to Context

The history array contains the full adventure record, but only a portion goes into the AI context. The system selects recent history that fits within context allocation (~50% of dynamic space).

Scripts can use history to access content that may have scrolled out of context.
