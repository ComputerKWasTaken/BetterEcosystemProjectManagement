# Multiplayer Overview

> AI Dungeon supports multiplayer adventures where multiple players collaborate in the same story.

## Overview

Multiplayer allows multiple players to participate in a single Adventure simultaneously. Players share the story, take turns (or act concurrently), and experience AI-generated content together. The host controls settings while guests can interact within those parameters.

## Starting a Multiplayer Game

### Create a New Game

1. Open Play menu
2. Select Multiplayer
3. Choose "Create Game"
4. Select scenario/settings
5. Game starts with you as host

### Get the Join Code

1. Open game menu (flame icon)
2. Select "Invite Players"
3. Share the 8-digit code with others

## Joining a Multiplayer Game

1. Open Play menu
2. Select Multiplayer
3. Enter the 8-digit join code
4. Select "Join Game"
5. You join as a guest

## Player Management

### Host Capabilities

The host can:
- Kick players (temporary removal, can rejoin)
- Block players (permanent removal, cannot rejoin)
- Control AI settings for everyone
- End the game

### Guest Capabilities

Guests can:
- Take actions in the story
- Edit their character name
- Leave the game voluntarily
- See all story content

## Character Names

Multiplayer typically uses Third Person mode:
- Each player enters a character name
- Actions show as "[Name] does X" instead of "You do X"
- Distinguishes between players for the AI

## AI Settings

The host's AI settings apply to everyone:
- Model selection
- Context length
- Advanced settings
- If host has premium, guests benefit from premium settings

This is important: a premium host can give free players access to premium model features.

## Player Indicators

In the game interface:
- Player avatars appear above the action bar
- Typing indicators show when someone is composing
- Actions are attributed to their source

## Player Limit

There's no strict player limit:
- The "(1/4)" suggestion is just a recommendation
- More players can join
- High player counts may cause chaotic overlapping actions

## Leaving vs. Exiting

Two ways to leave:
- **Leave**: Removes you from the multiplayer game; must rejoin with code
- **Exit Game**: Returns to home but keeps you in the game; can Continue later

## Use Cases

### Collaborative Storytelling

Friends building a story together, each controlling a character.

### Game Master Mode

One player guides/narrates while others play characters (manual DM style).

### Shared Experience

Watching friends play and joining in occasionally.

## Considerations

### Turn Management

No enforced turn order:
- Players can act anytime
- Actions may overlap
- Coordinate verbally for smoother experience

### Content Settings

Host's content settings apply:
- NSFW settings from host
- Guests operate under host's safety level
- Discuss preferences before playing

### Scenarios

Any scenario can be multiplayer:
- Third Person mode recommended
- Consider character-focused scenarios
- Some scenarios designed for multiplayer
